## 🌐 Overview

**Insight XR** is a next-generation STEM education platform that combines cutting-edge 3D visualization, artificial intelligence, and gamification to make learning physics, chemistry, biology, astronomy, and programming engaging and accessible for students of all levels.

### 🎯 Mission

To democratize quality STEM education by providing immersive, interactive learning experiences that transform complex scientific concepts into intuitive, visual understanding.

### 🌟 Why Insight XR?

- 🧠 **Visual Learning**: Complex concepts rendered in stunning 3D
- 🤖 **AI-Powered**: Personal AI tutor available 24/7
- 🎮 **Gamified**: Achievements, streaks, and progress tracking
- 📊 **Analytics**: Comprehensive learning analytics for students and teachers
- 🔐 **Secure**: Enterprise-grade security with Firebase
- 📱 **Responsive**: Works seamlessly on desktop, tablet, and mobile

## 🚀 Getting Started

### Prerequisites

- **Node.js** 18.0.0 or higher
- **npm** or **yarn**
- **Firebase Account** ([Sign up](https://firebase.google.com))
- **Google Gemini API Key** ([Get key](https://ai.google.dev))

### 📥 Installation

#### 1. Clone the Repository

```bash
git clone https://github.com/yourusername/insight-xr.git
cd insight-xr
```

#### 2. Install Dependencies

```bash
npm install
```

#### 3. Set Up Environment Variables

Create a `.env.local` file in the root directory:

```env
# Gemini AI API Key
GEMINI_API_KEY=your_gemini_api_key_here

# Firebase Configuration
FIREBASE_API_KEY=your_firebase_api_key
FIREBASE_AUTH_DOMAIN=your-project.firebaseapp.com
FIREBASE_PROJECT_ID=your-project-id
FIREBASE_STORAGE_BUCKET=your-project.appspot.com
FIREBASE_MESSAGING_SENDER_ID=123456789
FIREBASE_APP_ID=1:123456789:web:abc123
```

#### 4. Configure Firebase

Follow the detailed guide: [Firebase Setup Guide](insightxr-backend/docs/FIREBASE_SETUP.md)

**Quick setup:**
```bash
# Install Firebase CLI
npm install -g firebase-tools

# Login to Firebase
firebase login

# Initialize Firebase
firebase init

# Deploy security rules
firebase deploy --only firestore:rules,storage
```

#### 5. Run Development Server

```bash
npm run dev
```

Open [http://localhost:5173](http://localhost:5173) in your browser! 🎉

#### 6. Build for Production

```bash
npm run build
npm run preview
```

## 🔒 Security

### Implementation

#### Firebase Security Rules

**Firestore Rules**:
```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Users can only access their own data
    match /users/{userId} {
      allow read, write: if request.auth != null 
                        && request.auth.uid == userId;
    }
    
    // Teachers can view their students
    match /users/{userId} {
      allow read: if request.auth.token.role == 'teacher';
    }
    
    // Admins have full access
    match /{document=**} {
      allow read, write: if request.auth.token.role == 'admin';
    }
  }
}
```

**Storage Rules**:
```javascript
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /users/{userId}/{allPaths=**} {
      // 5MB file size limit
      allow write: if request.resource.size < 5 * 1024 * 1024
                   && request.resource.contentType.matches('image/.*')
                   && request.auth.uid == userId;
      allow read: if request.auth.uid == userId;
    }
  }
}
```

### Security Features
- ✅ Role-based access control
- ✅ Data encryption at rest and in transit
- ✅ Secure authentication with Firebase
- ✅ Environment variable protection
- ✅ Input validation and sanitization
- ✅ XSS and CSRF protection
- ✅ Rate limiting for API calls
